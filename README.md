Real-Time Traffic Simulation and Analytics System
Този проект представлява иновативна двукомпонентна система за симулация, събиране и обработка на уеб трафик в реално време. Цялата архитектура е напълно контейнеризирана чрез Docker и се оркестрира автоматично с Docker Compose.

Структура на проекта
Проектът е организиран по чист и модулен начин, разделящ логиката на приложението от инфраструктурните конфигурации:

traffic-analyzer/
│
├── generator/                 # Модул на персонализираната услуга
│   ├── app.py                 # Симулатор на трафик (Python)
│   ├── requirements.txt       # Зависимости на приложението
│   └── Dockerfile             # Инструкции за изграждане на Docker образ
│
├── compose.yml                # Docker Compose конфигурация за оркестрация
└── README.md                  # Документация на проекта

Как работи всеки от компонентите
Системата е изградена от две основни услуги (контейнери):

Traffic Generator (услуга traffic_generator):

Тип: Персонализирана услуга (наш сорс код).

Технология: Python 3.11 (лек slim образ).

Роля: Симулира реално потребителско поведение на уебсайт. На всеки 2 секунди генерира случайни метаданни за посещение: времево клеймо (timestamp), държава на произход, използван браузър, посетен URL адрес и HTTP статус код (напр. 200, 404, 500). Данните се сериализират в JSON формат и се изпращат към базата данни.

Data Store (услуга redis_db):

Тип: Готов официален образ от Docker Hub.

Технология: Redis 7 (минималистичен alpine образ).

Роля: Високопроизводителна NoSQL база данни, работещая в паметта (In-Memory). Тя приема потока от данни от генератора и ги структурира в бърза опашка (Redis List) под ключ site_traffic. Конфигурирана е да пази само последните 100 записа, за да оптимизира ресурсите.

Как е осъществена комуникацията между услугите
Комуникацията между контейнерите е реализирана чрез изолирана виртуална мрежа на Docker:

В compose.yml е дефинирана мрежа от тип bridge с име analyzer_network.

Двата контейнера са зачислени към тази мрежа, което ги изолира от външния свят и осигурява сигурност.

Благодарение на вградената DNS система на Docker, Python скриптът не използва статични IP адреси или localhost. Той се свързва с базата данни директно чрез нейното служебно име, дефинирано в Compose файла: redis_db на порт 6379.

Използвана е директивата depends_on, която гарантира, че базата данни ще стартира и ще бъде готова за работа преди Python генераторът да се опита да изпрати данни към нея.

Инструкции за изграждане и стартиране на контейнерите
За да стартирате и тествате проекта локално, следвайте тези стъпки:

Предварителни изисквания
Уверете се, че имате инсталиран и стартиран Docker Desktop.

Клониране на проекта
git clone https://github.com/вашето-потребителско-име/traffic-analyzer.git
cd traffic-analyzer

Стартиране на системата
Изпълнете следната команда в терминала си, за да компилирате нашия Dockerfile и да пуснете контейнерите в background режим:
docker compose up -d --build

(Флагът -d освобождава терминала ви, а --build подсигурява, че промените по Python кода ще бъдат отразени веднага.)

Проверка на логовете (Реално време)
За да се уверите, че генераторът работи и изпраща данни, можете да прегледате неговите логове:
docker logs -f traffic_gen_app

Проверка на записаните данни в Redis
За да докажете, че базата данни успешно съхранява информацията, можете да влезете в контейнера на Redis и да извлечете последните 5 записа директно през Redis CLI инструмента:
docker exec -it traffic_redis redis-cli LRANGE site_traffic 0 4

Спиране на системата
Когато приключите работа, можете да премахнете контейнерите и мрежата чрез:
docker compose down
